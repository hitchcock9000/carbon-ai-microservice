🧠 LSTM Deep Learning Model

Professor's Suggestion: Temporal Forecasting with Recurrent Networks

Architecture

  • Input: 24-hour lookback windows (sequences)
  • Layer 1: LSTM (128 units) + Dropout (0.2)
  • Layer 2: LSTM (64 units) + Dropout (0.2)
  • Layer 3: Dense (32 units, ReLU)
  • Output: Dense (1 unit) - next hour prediction

Training Strategy

  • Optimizer: Adam (lr=0.0001, gradient clipping)
  • Loss: MSE on log-transformed values
  • Callbacks: Early Stopping, ModelCheckpoint, ReduceLROnPlateau
  • Batch Size: 256 | Max Epochs: 50

🔧 Data Preprocessing Pipeline

1. Cleaning

  • Remove anomalies (meter_reading = 0)
  • Handle missing weather data
  • Interpolate temperature gaps

2. Feature Engineering

  • Log transformation (target)
  • Temporal features extraction
  • Degree hours calculation
  • Building aggregations

3. Scaling

  • MinMaxScaler for LSTM
  • Separate scalers for X and y
  • Preserve test distribution

4. Sequence Creation

  • Sliding windows (24 hours)
  • Shape: (samples, 24, features)
  • Maintains temporal order
